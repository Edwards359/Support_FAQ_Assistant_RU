# Публикация ассистента в GPTs Store

## Что должно быть готово

- Системный промпт из `system-prompt.md`.
- Файлы базы знаний из `knowledge/`.
- Результаты проверки по `tests/test-cases.md`.

## Шаги в OpenAI GPT Builder

1. Откройте GPT Builder: https://chat.openai.com/gpts/editor
2. Нажмите "Create".
3. В поле Instructions вставьте **актуальное** содержимое `system-prompt.md` (при каждом изменении файла в репозитории — снова вставить и нажать Update; автосинхронизации нет). Промпт задаёт тематику только поддержки, краткость ответов и правила безопасности.
4. В **Конфигурация** → **Возможности** (Capabilities) **отключите** **«Генерация изображений»** (*Image generation*), чтобы ассистент поддержки не создавал картинки. При необходимости отключите **«Холст»** (*Canvas*), если он не нужен.
5. В Knowledge загрузите файлы:
   - `knowledge/faq.md`
   - `knowledge/instructions.md`
   - `knowledge/templates.md`
6. Заполните имя, описание и подсказки для пользователей (имя GPT можно задать **Support FAQ Assistant RU** или вариант для вашего бренда).
7. Прогоните тестовые запросы из `tests/test-cases.md`.
8. Откройте настройки публикации:
   - Visibility: `Anyone with link` (открытый доступ по ссылке).
9. Нажмите Publish/Update.
10. Скопируйте публичную ссылку и передайте заказчику.

## Рекомендованное имя и описание GPT

- **Имя:** Support FAQ Assistant RU  
- **Описание:** Ассистент первой линии поддержки по FAQ и внутренним инструкциям: краткие пошаговые ответы на русском, цитаты источников, при необходимости — передача запроса оператору.

## Что передать заказчику после публикации

- Публичную ссылку на GPT.
- Краткую памятку: какие вопросы можно задавать.
- Контакт для эскалации сложных случаев.
