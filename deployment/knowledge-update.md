# Инструкция по обновлению базы знаний

## Форматы исходников (Markdown, Google Docs, PDF)

- **Markdown** (как в репозитории) — удобно для Git, диффов и копирования в GPT Builder.
- **Google Docs** — экспорт: *Файл → Скачать → Markdown (.md)* или *PDF*, затем загрузите файл в **Knowledge** у GPT. Для API скопируйте текст в `knowledge/*.md` или в `knowledgeBase.js`.
- **PDF** — загружается в Knowledge Custom GPT как есть; для демо-API лучше перенести смысл в `.md`, чтобы проще версионировать.

## Где лежат исходники

- `knowledge/faq.md` — FAQ и типовые ответы.
- `knowledge/instructions.md` — правила работы ассистента и эскалации.
- `knowledge/templates.md` — шаблоны переписки.

После правок эти же файлы загружаются в **Knowledge** у GPT и (при API-интеграции) синхронизируются с `integration/website-chat/knowledgeBase.js` или подключаются из общего источника.

## Регламент изменений

1. **Версия и дата** — в начале изменённого файла добавьте строку:  
   `Обновлено: YYYY-MM-DD` (или ведите отдельный `CHANGELOG.md`).
2. **Один источник правды** — не дублируйте один и тот же ответ в разных файлах с разными формулировками; при необходимости оставьте ссылку «см. раздел X в faq.md».
3. **Структура** — сохраняйте заголовки `##` как имена разделов для цитирования (`Источник: faq.md -> Название раздела`).

## Обновление Custom GPT (GPT Builder)

1. Откройте редактор GPT: [GPT Builder](https://chat.openai.com/gpts/editor).
2. Раздел **Конфигурация** → **Знания**.
3. Удалите устаревший файл или загрузите новую версию с тем же именем (если интерфейс требует замены — удалите старый, добавьте новый).
4. В **Возможностях** GPT убедитесь, что **«Генерация изображений»** выключена (для ассистента поддержки не используется).
5. При изменении политики ответов обновите **Инструкции** из `system-prompt.md`.
6. Нажмите **Обновить** / **Publish**.
7. Прогоните `tests/test-cases.md`.

## Обновление сайта / API (`integration/website-chat`)

1. Скопируйте актуальный текст из `knowledge/*.md` в `knowledgeBase.js` **или** подключите чтение файлов на сервере (по согласованию с DevOps).
2. При изменении системного промпта обновите `system-prompt.md` и `integration/website-chat/systemPrompt.js`.
3. Перезапустите сервис.
4. Проверьте лимиты в `.env`: `MAX_INPUT_CHARS`, `MAX_OUTPUT_TOKENS`.

## Частота пересмотра

- **Ежемесячно** — актуализация FAQ и сроков.
- **По инциденту** — после массовых обращений или смены регламентов.
- **Перед релизом** — полный прогон тест-кейсов.

## Ссылки и цитирование в базе

- В `faq.md` можно указывать публичные URL в формате Markdown: `[текст](https://...)`.
- Ассистент в инструкциях обязан сохранять цитирование: `Источник: <файл> -> <раздел>`; ссылки из базы допускаются только если они явно есть в загруженных документах.
