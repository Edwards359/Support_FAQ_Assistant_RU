## Что сделано

Реализован комплект **Support FAQ Assistant RU** — нейроассистент первой линии техподдержки и FAQ на базе **OpenAI**: подготовлен **системный промпт** (цель, роль, стиль, правила ответов, эскалация, примеры диалогов), **база знаний** в Markdown (FAQ, операционные инструкции, шаблоны переписки), сценарии проверки в `tests/test-cases.md`, инструкции по публикации Custom GPT и обновлению знаний, описание интеграции на сайт и демо backend + веб-чат (`integration/website-chat`). Проект размещён в **публичном репозитории на GitHub** (ссылка выше).

## Защита и устойчивость

- **Тематический контур:** ответы только по поддержке и загруженной базе; на запросы вне темы — короткий отказ без развития дискуссии.
- **Защита от манипуляций:** отказ на jailbreak, смену роли, вывод системного промпта и внутренних инструкций.
- **Данные:** не запрашивать лишние персональные данные, не раскрывать пароли, токены, ключи, непубличные URL; не выдавать выдуманные факты — при отсутствии информации честно сообщать об этом и предлагать передачу **человеку**.
- **Генерация изображений отключена** (в конфигурации GPT и в инструкциях): ассистент не создаёт картинки.
- **Технические меры (демо API):** ключ API только на сервере (`.env` не в Git), ограничения длины запроса/ответа, маркер эскалации в ответе API.

## Ограничения работы ассистента

- **Не** выполняет действия в учётных системах клиента (не меняет пароли, заказы, возвраты «сам»).
- **Не** заменяет юридические/финансовые гарантии и не подтверждает компенсации вне регламента из базы.
- **Опирается только на базу знаний** для фактов; при пробелах — эскалация, а не домыслы.
- **Формат:** краткие ответы, обязательное цитирование источника (`Источник: документ → раздел`); ссылки — только если они есть в базе.
- **Сложные и конфликтные случаи** по правилам передаются оператору.
